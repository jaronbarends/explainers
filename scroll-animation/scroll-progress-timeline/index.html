Using scroll timeline

Tie an animation to the scroll-progress of its nearest scrolling ancestor (in this case, the body) 

<body>
	<div class="progress-bar"></div>
	<div class="wrapper">...</div>
</body>


	@keyframes width-animation {
		from { transform: scaleX(0); }
		to { transform: scaleX(1); }
	}
	
	.progress-bar {
		position: sticky;
		top: 0;
		width: 100%;
	}

	.progress-bar::before {
		content: '';
		display: block;
		position: absolute;
		inset: 0;
		width: 100%;
		height: 100%;
		background: black;
		transform-origin: 0 0;
		animation: grow-progress linear;
		/* compare with normal animation - it just misses duration: */
		/* animation: grow-progress 3s linear; */
		animation-timeline: scroll(); /* by default, the nearest scroll container is used */
		/* animation-timeline: --page-scroll; */
	}

Section with overflow

It doesn't just work on the body, but on any scroll container. When you just use animation-timeline: scroll(), the elements nearest scroll-container is used (here it is this section) 

<section class="overflow-section">
	<div class="progress-bar"></div>
</section>
