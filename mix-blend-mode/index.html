mix-blend-mode: multiply

The mix-blend-mode CSS property sets how an element's content should blend with the content of the element's parent and the element's background. mix-blend-mode on MDN

element
child

You can think of this as if you have one image consisting of the element with its background and content, and one image consiting of the parent with its background and content (including any of the element's siblings that are on a lower plane).
Then the element-image is multiplied on the parent-image. There is no blending between the element itself and its content, nor with the parent and its content. 

.element {
	mix-blend-mode: multiply;
}

When the child element is white, its multiply blending effect with the parent will always be just the parent, making it look as if the child was cut out off the element.

When we make the element's background the same color and size as its parent, and remove the sibling element, it looks as if the child was cut out off the element. 

.parent {
	background: red;
	&::before { background: #fc0;/* yellow */ }
}
.element { background: inherit; }

This works especially nice with a black background. 

.parent {
background: black;
&::before { linear-gradient(to bottom right, red, blue); }
}
.element { background: inherit; }
.dot { background: white; }

This only works if the parent's (and thus the element's) background are darker than the parent's content, because then their color "wins" in the multiply blend. When the background is lighter, you'll just see the parent's content

background: #fc0;/* yellow */
&::before { background:  red;}
